# Ghost Yotei Phase B 外援提问稿 v2

本文档用于向 `Claude / Gemini / ChatGPT` 这类外部模型发起第二轮咨询。  
和上一版不同，这一版**不再主要问“根因是什么”**，而是问：

- 现在是不是该停止继续自动扩展
- 是否应该正式切换到“人工精修主导”的收口阶段
- 最终双语 `JSON` 的交付链路应该如何定义

这份文档分为两部分：

1. 给你自己看的简要说明
2. 一段可直接复制到外部模型对话框里的完整提问稿

---

## 1. 当前为什么要问第二轮外援

第一轮外援的核心价值是：

- 帮我们识别出 `Phase B` 的主问题不是 `OCR`
- 而是对齐单元设计错误
- 最终促成了从 `block-level` 切到 `cue-level + 1:N/N:1 + 局部窗口`

这一轮之后，工程已经继续推进到了一个新的阶段：

- 中文 `OCR` 已完成
- 英文 `OCR` 已完成
- `part01/part02` 已有比较稳的局部双语结果
- `part03/part04` 也各自拉出了 partial 成果
- 自动扩展的边际收益已经明显下降
- 人工收口入口已经工程化

所以现在更值得问外部模型的是：

- 还要不要继续自动补洞
- 还是应该正式切换成人工精修主导
- 最终交付链路应该如何从 `master + curation + context + review rounds` 收敛到“可交付双语 JSON”
